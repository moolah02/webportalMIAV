1. Overview & Base URL

The MIAV API is a RESTful API that accepts and returns JSON. All endpoints are prefixed with /api.

EnvironmentBase URL
Productionhttps://yourdomain.com/api
Developmenthttp://51.21.252.67:8080/api
All requests must include the Accept: application/json header. All write requests must include Content-Type: application/json.

2. Authentication

The API uses Bearer Token authentication via Laravel Sanctum. Obtain a token by calling POST /api/auth/login. Include the token in all subsequent requests:

Authorization: Bearer {your_token_here}

Tokens are revoked on logout. Tokens do not expire by default but can be refreshed.

3. Response Format

All API responses follow a consistent JSON structure:

Success Response

{
  "success": true,
  "message": "Optional message",
  "data": { ... }
}

Paginated Response

{
  "success": true,
  "data": [ ... ],
  "pagination": {
    "current_page": 1,
    "last_page": 5,
    "per_page": 15,
    "total": 75
  }
}

Error Response

{
  "success": false,
  "message": "Validation failed",
  "errors": {
    "field_name": ["Error description"]
  }
}

4. Authentication Endpoints

POST /auth/login

Authenticate a user and receive an API token. No auth required.

ParameterTypeRequiredDescription
emailstring✅Employee email address
passwordstring (min:6)✅Account password
{
  "success": true,
  "data": {
    "user": { "id": 1, "name": "John Doe", "email": "...", "role": "Technician", "permissions": [] },
    "token": "1|abc123...",
    "token_type": "Bearer"
  }
}

POST /auth/logout

Revoke the current token. Auth required.

GET /auth/user

Returns the authenticated user's profile. Auth required.

POST /auth/refresh

Revoke current token and issue a new one. Auth required.

GET /auth/profile

Returns full profile including permissions and avatar URL. Auth required.

PUT /auth/profile

Update profile fields. Auth required.

ParameterTypeRequiredDescription
namestring (max:255)❌Display name
phonestring (max:20)❌Phone number
avatarimage file (max:2MB)❌Profile photo
current_passwordstringIf changing passwordCurrent password for verification
new_passwordstring (min:6)❌New password (requires current_password)

POST /auth/forgot-password

Request a password reset email. No auth required.

ParameterTypeRequired
emailstring (email, must exist)✅
